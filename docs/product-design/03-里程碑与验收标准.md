# 03 里程碑与验收标准

## 3.1 阶段推进总则

- 先完成通用能力闭环，再进入模板化
- 没通过前置里程碑，不进入下一层能力

## 3.2 通用能力三大里程碑

### Milestone 1：可看、可跑、可留证据

验收点：

- `opskit install` 一键安装可用
- 页面可见 A~F 卡片（至少骨架）
- 状态来自 JSON（不是前端推导）
- 至少有 Preflight + Operate 报告

### Milestone 2：可自愈

验收点：

- 开机后可自动触发恢复
- 恢复具备就绪判定与有限重试
- 恢复失败有熔断，避免风暴
- 每次恢复有报告和证据

### Milestone 3：可验收交付

验收点：

- `opskit accept` 可生成验收包
- 包内含 hash/路径/状态等复核证据
- `opskit handover` 可汇总 A~F
- 页面可展示并下载最近交付产物

## 3.3 模板化里程碑（后续）

- Milestone 4：ELK 模板
- Milestone 5：OA（接管优先）
- Milestone 6：MinIO 模板

模板验收通则：

- 通过通用阶段验收
- 模板检查规则可复核
- 恢复策略可控（重试+熔断）
- 证据可审计（hash/路径/时间）
