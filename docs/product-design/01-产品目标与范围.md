# 01 产品目标与范围

## 1.1 产品目标

OpsKit 的目标是在离线/内网环境提供一套可交付、可审计的服务器生命周期运维底座，覆盖 `A -> F` 六阶段：

- A Preflight（接管检查）
- B Baseline（基线初始化）
- C Deploy（部署或接管声明）
- D Operate（巡检）
- E Recover（恢复）
- F Accept/Handover（验收与移交）

最终体验目标：

- 单二进制交付（`opskit`）
- 单命令安装（`opskit install`）
- 页面只读 JSON 展示状态
- 每个阶段都有报告和证据入口

## 1.2 核心原则

- 可重复：同阶段可重复执行，不破坏现状
- 可观察：页面与 JSON 状态一致
- 可证据化：输出报告、证据、交付包
- 可回溯：保留最近 N 次记录
- 可解释：失败有原因和建议动作

## 1.3 范围边界

OpsKit 负责：

- 生命周期编排与状态管理
- 通用检查、恢复、证据采集
- 模板驱动的业务栈扩展

OpsKit 不负责：

- 依赖外网下载
- 业务数据初始化（如建索引/建桶）
- 未声明的危险系统操作自动化执行

## 1.4 模板化策略（先通用后模板）

- 先完成通用能力（A/B/D/E/F）闭环
- 再增加模板（ELK/OA/MinIO）
- 模板只能“追加业务规则”，不能改写通用阶段基线语义
